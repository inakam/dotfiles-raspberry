# 日本語テキスト脱臭スキル

AI生成日本語に特有の「AI臭」を検出し、人間らしい自然な文章に変換する。

## ワークフロー概要

```
1. 対象テキストの取得
2. AI臭の検出・分析
3. 脱臭計画の策定
4. 段階的な脱臭の実施
5. 結果の検証
```

## ステップ1: 対象テキストの取得

### テキスト取得方法

- ユーザーがファイルパスを指定 → Readツールで読み込み
- ユーザーがテキストを直接貼り付け → そのまま分析
- 生成直後のテキスト → その場で脱臭

## ステップ2: AI臭の検出・分析

以下の5つのレイヤーでAI臭を検出する。詳細は[ai-smell-patterns.md](references/ai-smell-patterns.md)を参照。

### 検出対象のAI臭カテゴリ

| レイヤー           | 症状                                       | 優先度   |
| :----------------- | :----------------------------------------- | :------- |
| 視覚・構造         | 過剰な箇条書き、マークダウン記号の乱用     | Warning  |
| 統語・リズム       | 「です・ます」の定型反復、接続詞の過剰供給 | Critical |
| 語彙・文法         | 翻訳調表現、名詞化・サ変動詞の乱用         | Critical |
| 意味・語用         | 身体性と主観の欠如、両論併記の逃げ         | Warning  |
| 文化的コンテキスト | ハイコンテキストの無視、説明過多           | Info     |

### 分析レポート形式

```markdown
## AI臭分析レポート

**対象**: {テキスト名/ファイル名}
**文字数**: {総文字数}
**検出したAI臭**: {件数}

### Critical（必須修正）

- {箇所}: {AI臭の種類} - {説明}

### Warning（推奨修正）

- {箇所}: {AI臭の種類} - {説明}

### Info（改善提案）

- {箇所}: {AI臭の種類} - {説明}
```

## ステップ3: 脱臭計画の策定

検出したAI臭に対して、適用すべき脱臭パターンを計画する。
詳細は[deodorization-dictionary.md](references/deodorization-dictionary.md)を参照。

### 主要な脱臭手法

1. **語彙のリプレイス** - 翻訳調・硬い表現を日常語へ
2. **リズムの変調** - 文末・文長の意図的なバラつき
3. **ノイズの注入** - 人間らしい「揺らぎ」の演出
4. **身体性の付与** - 具体的な感覚描写の追加

## ステップ4: 段階的な脱臭

### 実施順序

1. Critical問題を優先的に修正
2. 文章全体のトーンを保ちながら修正
3. 必要に応じてユーザーに確認

### 4.1 語彙・表現の脱臭

```
# Before: 翻訳調・硬い表現
〜において → 〜で
〜を行う → 〜する
〜の実施を推奨します → 〜した方がいいですよ
〜の鍵となります → 〜が大事です
包括的な → まるっと、全部入りで
堅牢な → しっかりした
探求する → 考える、見ていく

# Before: 受動態・無生物主語
「この機能は、ユーザーに利便性を提供します。」
# After: 人間主語・能動態
「この機能を使えば、もっと便利になります。」

# Before: 名詞化
「課題の解決を行うことが重要です。」
# After: 動詞化
「課題を解決することが大事です。」
```

### 4.2 リズム・構文の脱臭

```
# Before: 単調な文末（です・ます連続）
「AIは便利です。多くの人が使っています。効率が上がります。」

# After: 文末のバリエーション
「AIは便利だ。多くの人が使っている。効率？ 確実に上がる。」

# 技法
- 体言止め: 「〜という特徴があります。」→「〜という特徴。」
- 倒置法: 「私はそれが好きです。」→「好きなんだよね、それが。」
- 極端な文長: 短文（10字以下）と長文（100字以上）を混在
```

### 4.3 接続詞の脱臭

```
# Before: 接続詞過多
「また、AIは便利です。さらに、効率も上がります。そして、コストも削減できます。」

# After: 接続詞削除（文脈で繋ぐ）
「AIは便利だ。効率も上がる。コストだって削減できる。」
```

### 4.4 身体性・主観の付与

```
# Before: 抽象的・他人事
「その知らせを聞いて、私はとても嬉しかった。」

# After: 身体反応を伴う描写
「その知らせを聞いた瞬間、思わずガッツポーズをしてしまった。心臓が早鐘を打っているのがわかった。」

# Before: 一方的な伝達
「これには注意が必要です。」

# After: 読者への問いかけ
「ここ、意外と落とし穴だと思いませんか？」
```

### 4.5 ノイズの注入

人間らしい「不完全さ」を演出する。

```
# 技法1: メタ発言の挿入
「…というわけで、効率化は重要です。（まあ、そう言いつつ、たまにはサボりたくなりますけどね）」

# 技法2: 脱線・余談
「少し話が逸れますが、」

# 技法3: 本音・愚痴
「正直、最初は信じていなかった。」
```

### 4.6 過剰な配慮の削除

```
# Before: 過剰なDisclaimer
「多様な視点がありますが、一部の人には不快かもしれませんが、」

# After: 削除または簡潔に
（不要なら削除）

# Before: 断定の回避
「〜と言えるでしょう」「〜の可能性があります」

# After: 責任を持って断定
「〜だ」「〜である」
```

## ステップ5: 結果の検証

### 脱臭チェックリスト（7つのツール）

- [ ] **リズム確認**: 「〜です/〜ます」が3回以上続いていないか
- [ ] **接続詞ダイエット**: 文頭の「また」「さらに」「そして」を削除しても通じるか
- [ ] **日常語への翻訳**: 「〜において」「〜を行う」が残っていないか
- [ ] **一次情報の注入**: 「私」の具体的な体験や失敗談が入っているか
- [ ] **具体化**: 抽象概念を具体的なシーンに言い換えているか
- [ ] **ノイズの許容**: きれいすぎず、感情的な吐露が含まれているか
- [ ] **温度感**: 読んでいて書き手の「顔」が浮かぶか

### 脱臭完了レポート

```markdown
## 脱臭完了レポート

**対象**: {テキスト名}
**Before**: {修正前文字数}字
**After**: {修正後文字数}字

### 実施した修正

1. {修正内容}
2. {修正内容}

### 脱臭効果

- 翻訳調表現: {X}箇所を修正
- リズム改善: 文末バリエーション追加
- 身体性付与: 具体的描写を{Y}箇所追加
```

## プロンプトエンジニアリングによる予防

生成前に以下の指示をプロンプトに含めることで、AI臭を予防できる。

### ペルソナの詳細設定

```
あなたは現場叩き上げの40代マーケターです。
教科書的な理論よりも、泥臭い現場の数値を信じています。
最近のAIブームには少し懐疑的ですが、便利なものは使う主義です。
若手に向けて、居酒屋で語るような、少し砕けた口調で書いてください。
```

### 禁止指示（ネガティブプロンプト）

```
以下を禁止します：
- 文末が「〜です」「〜ます」で終わる文が2回以上続くこと
- 「また」「さらに」「そして」を文頭で使用すること
- 「〜において」「〜を行う」「〜の実現」「包括的」「堅牢な」「探求する」を使用すること
- 文頭の絵文字（✅、🚀など）
- コロン（：）を使った導入
```

### バースト性の注入指示

```
一文の長さを意図的にバラつかせてください。
20文字以下の短い文と、80文字以上の長い文を交互に織り交ぜてください。
時には体言止めや、主語のない文を使っても構いません。
```

### Few-Shotプロンプティング（文体模写）

ユーザーの文体サンプルを元に、AIにスタイルを模倣させる手法。言葉で説明するより実例を見せる方が精度が高い。

**使用手順**:

1. AskUserQuestionツールでユーザーに文体サンプルを質問する

```
理想とする文体のサンプルテキストを3〜5例お持ちですか？
過去に自分が書いた文章や、お手本にしたい文章があれば貼り付けてください。
これを元にAIの出力スタイルを調整できます。
```

2. サンプルを受け取ったら、以下のプロンプト構造で生成を指示

```
以下の【文体サンプル】のトーン、リズム、語彙の選び方、改行のタイミングを分析し、
それを忠実に再現して、新しいテーマ「{テーマ}」について書いてください。

【文体サンプル】
{ユーザーが提供したサンプル}
```

**効果**: 論理構造だけでなく、書き手の「愚痴っぽさ」「息遣い」まで学習し、出力に反映させることができる。

## ケーススタディ

### ケース1: 謝罪メールの脱臭

**【AI生成文（Before）】**

```
平素よりお世話になっております。この度は、弊社のシステム障害により
多大なるご迷惑をおかけし、深くお詫び申し上げます。現在は復旧して
おりますが、原因究明と再発防止に努めてまいる所存です。何卒ご容赦
いただけますようお願い申し上げます。
```

**診断**: 完璧だが心がこもっていない。「コピペ」に見える。どこにも「私」の痛みがない。

**脱臭アプローチ**: 具体的な状況描写、書き手の主観的な焦り、定型句の破壊。

**【脱臭後の文章（After）】**

```
平素よりお世話になっております。
この度は、弊社システム障害により、〇〇様の業務を止めてしまう事態と
なり、本当に申し訳ございませんでした。
画面が動かなくなった際、どれほどご不安だったかと想像すると、
お詫びの言葉も見つかりません。
先ほど復旧はいたしましたが、二度とこのようなご迷惑をおかけしないよう、
私自身が責任を持って原因を徹底的に洗い出します。
まずはメールにて恐縮ですが、深くお詫び申し上げます。
```

**修正ポイント**:

- 「〇〇様」と個別化（対話感）
- 「どれほどご不安だったか」と相手の感情に言及（身体性）
- 「私自身が責任を持って」と主語を明確化（主観）
- 定型句「何卒ご容赦」を具体的な行動に変換

### ケース2: 商品レビュー（ガジェット）の脱臭

**【AI生成文（Before）】**

```
この新型イヤホンは、ノイズキャンセリング機能が優れています。音質も
クリアで、低音から高音までバランスよく再生されます。通勤通学に最適
です。デザインも洗練されており、所有欲を満たしてくれるでしょう。
```

**診断**: カタログスペックの羅列。「所有欲を満たしてくれるでしょう」というAI特有の上から目線/推測表現が鼻につく。

**脱臭アプローチ**: 「私」の主語化、五感の描写、ネガティブな本音（リアリティ）。

**【脱臭後の文章（After）】**

```
ノイキャン、正直ナメてました。
カフェでスイッチを入れた瞬間、周りの雑音が「フッ」と消えて、
自分だけ真空パックされたかのような静寂。これ、一度味わうと戻れないかも。
音質は、ドンシャリというよりは、ボーカルが耳元で囁くような近さを感じます。
ただ、ケースに指紋がつきやすいのが地味に気になるかな。
まあ、それを補って余りある没入感なんですけどね。
通勤電車がこれのおかげで「書斎」に変わりました。
```

**修正ポイント**:

- 「ナメてました」と本音から入る（ノイズ）
- 「フッと消えて」「真空パック」と擬音・比喩（身体性）
- 「地味に気になる」とネガティブも正直に（リアリティ）
- 「〜でしょう」を「〜かも」「〜なんですけどね」に変換（口語化）

## 注意事項

- 意味の変更は最小限に（脱臭のみ）
- 専門用語は文脈に応じて維持
- フォーマルさのレベルはユーザーに確認
