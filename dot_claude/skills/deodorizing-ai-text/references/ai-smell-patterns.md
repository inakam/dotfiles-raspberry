## 1. 視覚的・構造的レイヤー

### 1.1 箇条書きへの執着

**症状**: 情緒的なエッセイやストーリーでも内容を分解し箇条書きに整形する

**違和感の理由**: 日本語の散文では論理は文と文の接合の中に埋め込まれることが好まれる。箇条書きは「仕様書」の形式であり、読み手に「対話」ではなく「通達」を感じさせる

**例**:

```
# AI生成（違和感あり）
結論は以下の3点です：
- 効率が上がる
- コストが下がる
- 品質が向上する

# 人間らしい表現
効率が上がって、コストも下がる。おまけに品質まで良くなる。
```

### 1.2 マークダウン記法の不自然な使用

| 記号・記法   | AIの使用例                     | 違和感の理由                                 |
| :----------- | :----------------------------- | :------------------------------------------- |
| コロン（：） | ポイントは以下の通りです**：** | 英語 "Here is:" の直訳。日本語では句点で繋ぐ |
| 太字強調     | ここで重要なのは**共感性**です | SEO対策的。私信や報告書には不自然            |
| 文頭絵文字   | ✅ 効率化 🚀 成長              | 海外SaaSブログのスタイル。日本の文脈では浮く |

## 2. 統語的・リズム的レイヤー

### 2.1 「です・ます」の定型反復

**症状**: 文末が「〜です」「〜ます」で3回以上連続する

**違和感の理由**: 予測可能性が高まり、読者は無意識に退屈を感じる。音楽で全ての小節が同じコード進行で終わるようなもの

**例**:

```
# AI生成（違和感あり）
AIは便利です。多くの人が使っています。効率が上がります。コストも下がります。

# 人間らしい表現
AIは便利だ。多くの人が使っている。効率？ 確実に上がる。コストだって下がる。
```

### 2.2 接続詞の過剰供給

**症状**: 「また」「さらに」「そして」「しかし」「一方で」がほぼ全文頭に出現

**違和感の理由**: 日本語はハイコンテキスト言語。論理関係は行間で察することが好まれる。過剰な接続詞は読者の読解力を信頼していない「くどさ」を生む

**例**:

```
# AI生成（違和感あり）
また、AIは効率的です。さらに、コストも削減できます。そして、品質も向上します。しかし、課題もあります。一方で、解決策も存在します。

# 人間らしい表現
AIは効率的だ。コストも削減できる。品質だって向上する。ただ、課題はある。解決策？ ちゃんとある。
```

## 3. 語彙・文法的レイヤー

### 3.1 無生物主語と受動態の多用

**症状**: 英語の "The tool helps you." を直訳した無生物主語構文

**例**:

```
# AI生成（違和感あり）
この機能は、ユーザーに利便性を提供します。
このツールは、作業効率を向上させます。

# 人間らしい表現
この機能を使えば、もっと便利になります。
このツールを使うと、作業が楽になる。
```

### 3.2 名詞化（Nominalization）とサ変動詞の乱用

**症状**: 動詞で語るべきところを「名詞＋行う」形式に変換

**例**:

```
# AI生成（違和感あり）
課題の解決を行うことが重要です。
議論の実施を推奨します。
分析の実行が必要です。

# 人間らしい表現
課題を解決することが大事です。
議論したほうがいいですよ。
分析しないといけない。
```

### 3.3 直訳由来の特有語彙

| AI頻出語彙 | 推定される英語 | 違和感の理由                         |
| :--------- | :------------- | :----------------------------------- |
| 鍵となる   | Key            | 「重要」「ポイント」で良い文脈で多用 |
| 探求する   | Explore        | 「見ていく」「考える」が自然         |
| 革命的な   | Revolutionary  | 乱発で陳腐化                         |
| 包括的な   | Comprehensive  | 「まるっと」「全部入り」が自然       |
| 堅牢な     | Robust         | セキュリティ以外では一般的でない     |
| 〜において | In             | 「〜で」で済む場合がほとんど         |

## 4. 意味論的・語用論的レイヤー

### 4.1 クオリアと一次情報の欠落

**症状**: 五感を通じた経験の具体的描写がない

**例**:

```
# AI生成（違和感あり）
美味しいカレーを食べました。適度な運動は健康に良いです。

# 人間らしい表現
舌を刺すようなスパイスの刺激が忘れられない。スクワットの翌日、階段を降りる時のあの独特の辛さ。
```

### 4.2 「神の視点」と優等生バイアス

**症状**: どの立場にも偏らない、摩擦を恐れる優等生的な文章

**例**:

```
# AI生成（違和感あり）
メリットもありますが、デメリットもあります。バランスが大切です。
〜と言えるでしょう。〜の可能性があります。

# 人間らしい表現
正直、デメリットもある。でも、メリットの方がデカい。
〜だ。間違いない。
```

## 5. 文化的コンテキストレイヤー

### 5.1 ハイコンテキストの無視

**症状**: すべてを言語化して説明しようとするローコンテキストなスタイル

**違和感の理由**: 行間を読むことを美徳とする日本の読者には「説明過多」「野暮」と映る

### 5.2 ポライトネスの不自然さ

**症状**: 文法的に正しいが、相手との距離感を測り損ねた敬語

**違和感の理由**: 「慇懃無礼」「他人行儀」な印象。社会的文脈の理解不足
