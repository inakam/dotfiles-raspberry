## 1. 視覚的物理学

### スマホ3行の法則

| 特徴     | 一般的スタイル   | けんすうスタイル         |
| :------- | :--------------- | :----------------------- |
| 段落長   | 5〜10行          | **1〜3行（40〜80文字）** |
| 改行頻度 | 論理の区切りのみ | 視覚的リズムで改行       |
| 空白行   | 行間なし         | 積極的な余白活用         |

### 余白（マ）の配置

重要な結論や転換点の**直前**に空行を入れる。

```
しかし、ここで問題が起きます。

（空行で視線を止める）

実は、○○だったんです。
```

### 視覚的アンカー

- 記事の2割しか読まない読者を前提に設計
- **太字だけ読めば大意がつかめる**構成
- 3つ以上の並列要素は必ず箇条書き

## 2. 統語論的エンジニアリング

### 一文一義の原則

日本語の「〜が、〜ので、〜して」チェーンを断ち切る。

| 修正前                                                                                                                       | 修正後                                                                                                                                                     |
| :--------------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 「僕は起業家として多くの失敗をしてきましたが、そこから学んだことは多く、特に物語思考という考え方は役に立つので紹介します。」 | 「僕は起業家として、多くの失敗をしてきました。しかし、そこから学んだことも多いです。特に『物語思考』という考え方は役に立ちます。今回はそれを紹介します。」 |

### 接続詞の処理ルール

| 接続詞                 | 処理                         |
| :--------------------- | :--------------------------- |
| しかし、とはいえ、でも | **残す**（論理転換シグナル） |
| また、さらに、そして   | **削除**（文脈で自明）       |
| だから、なぜなら       | 適宜残す                     |

### 主語と述語の近接配置

```
# 悪い例（主語と述語が遠い）
「Aという理由で、Bという背景がありながらも、CさんはDした」

# けんすう風（結論先行）
「CさんはDしました。理由はAです。背景にはBがありました」
```

## 3. 文末バリエーション

### です/ます連続の回避

3連続以上の「です/ます」を避ける。

| テクニック | 例                           |
| :--------- | :--------------------------- |
| 体言止め   | 「〜という特徴。」           |
| 倒置法     | 「好きなんだよね、それが。」 |
| 問いかけ   | 「〜ですよね？」             |
| 感嘆       | 「〜なんです！」             |

### 語りかけ表現

```
〜です。 → 〜なんです。
〜です。 → 〜ですよね。
〜です。 → 〜かもしれません。
〜です。 → 〜だと思います。
```

## 4. トーン＆マナー

### 導入パターン

```
こんにちは！

今日は○○について書いてみます。

結論から言うと、××です。
```

### 脆弱性の開示（Vulnerability）

自分の弱みや失敗談を共有することで、読者の防御壁を下げる。

```
# 効果的な開示例
- 「30代を超えると徹夜ができなくなる」
- 「運動を習慣化するのに15年かかった」
- 「僕は糖尿病なので…」
- 「片付けができないから…」
```

### 乾いた（Cool）視点

感情に溺れず、「メタ視点の自分」として客観的に記述。

```
# Before（湿った文体）
「あの時、私は本当に悔しくて、涙が止まらなくて…」

# After（乾いた文体）
「正直、めちゃくちゃ悔しかったです。でも、今振り返ると、あれがきっかけで変われたんですよね。」
```

## 5. 構成パターン

### PREP法の徹底

1. **Point（結論）**: 最初に結論を述べる
2. **Reason（理由）**: なぜそうなのかを説明
3. **Example（具体例）**: 自分の経験や具体的な事例
4. **Point（結論の再強調）**: 最後にもう一度結論

### 未来逆算型の構成

```
# 過去積み上げ型（避ける）
Aをして → Bをして → 結果Cになった

# 未来逆算型（推奨）
Cになりたい → そのためにBが必要 → だから今はAをする
```

## 6. アンチパターン（避けるべきもの）

### 翻訳調表現

| 避ける               | 使う                 |
| :------------------- | :------------------- |
| 〜において           | 〜で                 |
| 〜を行う             | 〜する               |
| 〜の実施を推奨します | 〜した方がいいですよ |
| 〜の鍵となります     | 〜が大事です         |
| 包括的な             | まるっと             |
| 堅牢な               | しっかりした         |
| 探求する             | 考える、見ていく     |

### 過剰な配慮

```
# Before（過剰なDisclaimer）
「多様な視点がありますが、一部の人には不快かもしれませんが…」

# After（削除または簡潔に）
（不要なら削除）
```

### 断定の回避（曖昧表現）

```
# Before
「〜と言えるでしょう」「〜の可能性があります」

# After
「〜です」「〜だと思います」（責任を持って断定）
```
