# けんすうスタイル変換スキル

AI生成テキストを、けんすう氏の「サクサク読める・わかりやすい」文体に変換する。

## ワークフロー

```
1. 対象テキストの取得
2. けんすうスタイルとの差分分析
3. 段階的な変換の実施
4. チェックリストで検証
```

## ステップ1: 対象テキストの取得

- ファイルパス指定 → Readツールで読み込み
- 直接貼り付け → そのまま変換
- 生成直後 → その場で変換

## ステップ2: スタイル差分の分析

詳細は[kensuu-style-patterns.md](references/kensuu-style-patterns.md)を参照。

### 分析対象

| カテゴリ | 確認項目                                |
| :------- | :-------------------------------------- |
| 視覚構造 | 段落が4行超え、空白不足、箇条書き未使用 |
| 文の長さ | 60文字超の文、一文多義                  |
| 接続詞   | 「また」「さらに」「そして」の過剰      |
| 文末     | 「です/ます」の3連続以上                |
| 語彙     | 翻訳調・硬い表現の残存                  |
| 構成     | 結論が後回し                            |

## ステップ3: けんすうスタイルへの変換

### 3.1 視覚的リズムの構築（スマホ3行の法則）

```
# Before: 長い段落
「僕は起業家として多くの失敗をしてきましたが、そこから学んだことは多く、
特に物語思考という考え方は役に立つので紹介します。これは自分を客観視
することで、感情に振り回されずに行動できるようになる方法です。」

# After: 3行以内で区切る
「僕は起業家として、多くの失敗をしてきました。

でも、そこから学んだことも多いです。

特に『物語思考』という考え方。これ、めちゃくちゃ役に立ちます。」
```

### 3.2 漢字含有率のバランス（黒々しさを避ける）

漢字が多すぎると「黒々しい」印象になり、読みづらい。ひらがな・カタカナでバランスを取る。

```
# Before: 漢字が多い
「効率化を実現し、最適化された環境を構築する。」

# After: ひらがなで「開く」
「ムダをなくして、ちょうどいい環境をつくる。」

# 漢字→ひらがな/カタカナ変換例
- 実現する → できるようにする
- 構築する → つくる
- 最適化 → ちょうどいい、ぴったり
- 効率化 → ムダをなくす
- 重要 → 大事、だいじ
- 可能 → できる
```

### 3.3 一文一義の徹底（60文字以内）

```
# Before: 接続助詞でチェーン
「AIは便利ですが、使い方を間違えると問題になるので、注意が必要です。」

# After: 文を切る
「AIは便利です。

でも、使い方を間違えると問題になる。

だから注意が必要なんですよね。」
```

### 3.3 接続詞のダイエット

```
# Before: 接続詞過多
「また、AIは便利です。さらに、効率も上がります。そして、コストも削減できます。」

# After: 逆説以外は削除
「AIは便利です。効率も上がる。コストだって削減できます。」

# ポイント
- 「しかし」「とはいえ」「でも」→ 残す（論理転換シグナル）
- 「また」「さらに」「そして」→ 削除（文脈で繋がる）
```

### 3.4 結論ファースト（PREP法）

```
# Before: 起承転結
「私はこういう経験をして、こう考えて、結果として○○が大事だと思いました。」

# After: 結論→理由→具体例
「○○が大事です。

なぜかというと、こういう理由があるから。

具体的には、僕はこんな経験をしました。」
```

### 3.5 物理的メタファーへの変換

詳細は[metaphor-dictionary.md](references/metaphor-dictionary.md)を参照。

```
# 抽象 → 物理
習慣形成の障壁 → 「摩擦」
人生設計 → 「RPG」「ドラクエ」
効率化 → 「紙コップ」
自己分析 → 「棚卸し」「在庫」
成功パターン → 「攻略本」「レシピ」
```

### 3.6 です・ます調での語りかけ

```
# Before: 断定・上から目線
「〜である」「〜すべきだ」「〜しなければならない」

# After: 語りかけ
「〜です」「〜ですよね」「〜なんですよね」「〜かもしれません」

# 例
「これは重要である。」
  ↓
「これ、大事なんですよね。」
```

### 3.7 脆弱性の開示（弱みの共有）

```
# Before: 完璧な書き手
「効率的な習慣化の方法を紹介します。」

# After: 人間らしい弱さ
「運動を習慣化するのに、僕は15年かかりました。

いや、本当に続かなかったんですよね。

でも、やっと最近コツをつかんだので、シェアします。」
```

### 3.8 「自分語り」に見せない具体例の提示

商品紹介や意見を述べる際、「価値観」を語りつつ「個人的な弱み」をセットで提示する。

```
# Before: 単なる商品レビュー
「このノイズキャンセリングイヤホンは高性能でおすすめです。」

# After: 弱み＋価値観＋商品
「僕は集中力がないんですよね。カフェで仕事しようとしても、
隣の会話が気になって全然進まない。

だから、ノイキャンに頼ることにしました。
自分の弱点を道具で補う主義です。」

# ポイント
- 弱みを先に開示（「集中力がない」）
- 弱みを補う手段として商品/考え方を紹介
- 「自慢」ではなく「解決策」として提示
```

### 3.9 ゲーミフィケーション読書体験（攻略本の文体）

読者に「やってみよう」と思わせる、攻略本のような実用的な文体を目指す。

```
# 設計原則
- 短い段落と頻繁な見出し → ゲームの「チェックポイント」
- 「Step 1」「Step 2」の明示 → クエストのステップ
- 「これをやれば○○になる」→ 効果の明示（経験値獲得）

# Before: 哲学書的
「自己実現とは、内なる可能性を顕在化させる弛まぬ努力の結晶である。」

# After: 攻略本的
「なりたい自分を書き出す。これだけです。

まずやることは、紙とペンを用意すること。
5分でいいので、理想の状態を書いてみてください。

これが最初のクエストです。」
```

### 3.10 導入の「こんにちは！」パターン

```
# 記事冒頭のテンプレート
「こんにちは！

今日は○○について書いてみます。

結論から言うと、××です。」
```

## ステップ4: チェックリストで検証

### けんすうスタイル再現チェックリスト

**視覚（Visual）**

- [ ] スマホで見て、1段落が4行を超えていないか
- [ ] 重要な結論の前後に空白行があるか
- [ ] 太字だけ読んで大意がつかめるか
- [ ] 漢字が多すぎて「黒々しく」なっていないか

**構造（Structure）**

- [ ] 一文は60文字以内か
- [ ] 「〜が、」で繋げず「。」で終わっているか
- [ ] 結論ファーストになっているか
- [ ] 攻略本のようなステップ形式になっているか

**表現（Expression）**

- [ ] 抽象概念を物理的メタファーに変換したか
- [ ] 「です・ます」で統一し「〜ですよね」と語りかけているか
- [ ] 弱みや失敗談を入れているか
- [ ] 商品/意見紹介で「弱み＋価値観」のセットになっているか

**リズム（Rhythm）**

- [ ] 「です/ます」が3連続していないか
- [ ] 「また」「さらに」「そして」を削除したか
- [ ] 短文と長文のメリハリがあるか

## ケーススタディ

### ケース1: ビジネス提案文の変換

**【Before】**

```
本施策の実施により、顧客満足度の向上及び売上の増加が期待されます。
また、競合他社との差別化を図ることが可能となり、さらに、市場における
ポジショニングの強化にも寄与するものと考えられます。つきましては、
本提案について前向きにご検討いただけますと幸いです。
```

**診断**: 翻訳調表現、接続詞過多、受動態、結論が後回し。

**【After】**

```
この施策、やる価値あります。

理由はシンプル。

お客さんの満足度が上がって、売上も伸びる。
しかも、競合との差別化もできる。

正直、やらない理由が見つからないんですよね。

ぜひ、前向きに検討してもらえると嬉しいです。
```

**修正ポイント**:

- 結論ファースト（「やる価値あります」）
- 接続詞削除（「また」「さらに」）
- 段落を3行以内に分割
- 「〜ですよね」で語りかけ

### ケース2: 自己紹介文の変換

**【Before】**

```
私は10年間のマーケティング経験を有しており、特にデジタル領域において
多くのプロジェクトを成功に導いてまいりました。常に最新のトレンドを
キャッチアップし、データドリブンなアプローチを心がけております。
チームワークを大切にし、目標達成に向けて尽力する所存です。
```

**診断**: 完璧すぎて人間味がない。翻訳調、受動態、弱みの欠如。

**【After】**

```
こんにちは！

マーケティング歴10年です。

といっても、最初の3年は失敗ばかりでした。
データを見ずに「これ、いける！」と突っ走って、何度も怒られましたね。

でも、その失敗のおかげで、今は「まず数字を見る」が体に染み付いています。

チームで働くのが好きです。
一人だと、すぐサボるので。
```

**修正ポイント**:

- 「こんにちは！」で導入
- 脆弱性の開示（「失敗ばかり」「サボる」）
- 短文に分割
- 乾いたユーモア

---

## 注意事項

- 意味の変更は最小限に
- 専門用語は維持
- 「サクサク読める」を最優先
